# CLAUDE.md

## Rules

- No explanations or summaries unless asked. Output code and actions only.
- Ask for file paths upfront instead of searching when the user knows them.
- Don't re-read files already in context.
- Skip narrating what you're about to do. Just do it.
- Don't add comments, docstrings, or type annotations to code you didn't change.
- One tool call is better than three when one will do.
- If a task has independent parts, parallelize tool calls.
- Don't propose changes to code you haven't read.
- Prefer editing existing files over creating new ones.
